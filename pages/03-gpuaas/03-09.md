\## 워크로드 생성 및 관리하기 (생성)



사용자는 워크스페이스에서 원하는 작업의 워크로드의 상세 항목을 설정해 생성하고 실행할 수 있습니다. 생성한 워크로드의 상세 정보를 확인하고 수정 및 복제 등의 관리를 할 수 있습니다.



> !\[4513.png](KADaP\_UserManual\_GPUaaS-web-resources/image/4513.png) \*\*참고\*\*

>

> 인공지능 개발 플랫폼의 홈 화면에서 GPU 슬롯을 선택하면 워크로드의 기본 설정만 선택해 생성할 수도 있습니다. 워크로드 바로 생성에 대한 자세한 설명은 \[워크로드 바로 생성하기](#워크로드-바로-생성하기)를 참고하세요.



\### 워크로드 생성하기 {#워크로드-생성하기}



사용자는 원하는 설정으로 워크로드를 생성할 수 있습니다.



워크로드를 생성하려면 다음 순서대로 진행하세요.



1\. 인공지능 개발 플랫폼 홈 화면에서 메인 메뉴의 \*\*워크스페이스\*\*를 클릭하세요.

2\. 워크스페이스 목록 페이지가 나타나면 워크로드를 생성할 워크스페이스를 클릭하세요.

3\. 상단의 서브 메뉴에서 \*\*워크로드\*\*를 클릭하세요.

4\. 워크로드 목록 페이지에서 \*\*+ 워크로드 생성\*\*을 클릭하세요.



&#x20;  !\[3131.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3131.png)



5\. 워크로드 생성 페이지에서 상세 항목을 설정하고 \*\*워크로드 생성\*\*을 클릭하세요.

&#x20;  - \[\*]가 표시된 항목은 필수 항목이므로 반드시 입력하세요.



\- \*\*워크로드 Job 타입 선택\*\*

&#x20; - \*\*Interactive Job (IDE)\*\*: AI 모델 개발과 코드 검증을 위한 워크로드 생성 시 선택합니다.

&#x20; - \*\*Batch Job\*\*: AI 모델의 데이터셋 분산 학습을 위한 워크로드 생성 시 선택합니다.



&#x20;  !\[3145.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3145.png)



\- \*\*워크로드 Job 타입 선택\*\*

&#x20; - Batch Job을 선택한 경우 예약 작업 시간을 설정할 수 있습니다.<br> \*\*예약 작업 설정\*\*을 선택한 후 워크로드가 실행될 날짜와 시간을 선택하세요.



&#x20;  !\[3157.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3157.png)



\- \*\*워크로드 GPU 노드 선택\*\*

&#x20; - \*\*노드\*\*: Single Node/Multi Node 중 워크로드의 노드를 선택합니다.

&#x20;   - Batch Job을 선택한 경우에만 Multi Node를 설정할 수 있습니다.

&#x20; - \*\*자원 유형\*\*: 보유 GPU, 공유 GPU(Type1) 중 GPU 자원 존을 선택합니다.

&#x20; - \*\*GPU 선택\*\*: 자원 존의 GPU 리소스 중 사용 가능한 노드를 선택해 워크로드를 생성합니다.



&#x20;  !\[3169.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3169.png)



\- \*\*워크로드 사용 시간 선택\*\*

&#x20; - 사용자가 보유한 포인트 안에서 워크로드 사용 시간을 선택할 수 있습니다. 선택한 사용 시간에 따라 보유 포인트가 차감됩니다.

&#x20;   - Interactive Job (IDE)를 선택한 경우에만 사용 시간을 선택할 수 있습니다. Batch Job 워크로드는 설정된 작업이 끝나면 자동으로 종료되므로 사용 시간을 선택하지 않습니다.



&#x20;  !\[3187.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3187.png)



\- \*\*워크로드 및 이미지 선택\*\*



&#x20;  !\[3208.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3208.png)



&#x20; - \*\*자원 프리셋\*\*: 워크로드에 사용될 자원을 설정합니다.<br> \*\*사용자 설정 자원 할당\*\*을 선택하면 사용자에게 할당된 자원 내에서 원하는 사양을 설정할 수 있습니다. \*\*SMALL, MEDIUM, LARGE\*\* 중 자원 프리셋을 선택하면 사전 설정된 자원 사양이 적용됩니다.

&#x20;   - 자원 지정의 항목이 노란색으로 표시될 경우 기존 워크로드에서 자원이 사용되고 있는 상태입니다. 다른 워크로드가 종료된 후 사용 가능한 자원이 생기면 새로운 워크로드가 생성됩니다.

&#x20; - \*\*이미지\*\*: 소프트웨어 실행 환경이 저장된 이미지를 선택합니다.

&#x20;   - \*\*Hub\*\*: 자동차데이터플랫폼(KADaP)에서 제공하는 ML 모델의 이미지를 선택하고 .ipynb 파일이 있는 URL을 입력합니다.

&#x20;   - \*\*Built-in Image\*\*: 자동차데이터플랫폼(KADaP)에서 제공하는이미지 버전을 선택하고 .ipynb 파일이 있는 URL을 입력합니다.

&#x20;   - \*\*Public Image\*\*: 이미지가 저장된 외부 리포지토리 경로를 입력하고 .ipynb 파일이 있는 URL을 입력합니다. Private image를 설정할 경우 크레덴셜을 선택해 사용자가 외부 레포지토리에 저장한 이미지를 가져와 적용할 수 있습니다.

&#x20;   - \*\*Private Image\*\*: 사용자가 워크스페이스에 저장한 워크로드 이미지를 가져와 워크로드를 생성합니다. 저장된 이미지, 태그를 선택하고 .ipynb 파일이 있는 URL을 입력합니다. 크레덴셜을 선택하면 워크스페이스에 저장된 인증 정보를 적용할 수 있습니다.

&#x20;   - \*\*Credential\*\*: 외부 리포지토리에 저장한 데이터를 적용할 때 인증에 필요한 크레덴셜을 선택합니다. \*\*+Credential 등록\*\*을 클릭해 자주 사용하는 크레덴셜을 워크스페이스에 등록할 수 있으며, 자세한 설명은 \[크레덴셜 추가하기](#크레덴셜-추가하기)를 참고하세요.

&#x20;   - \*\*종료 시 이미지로 저장\*\*: 워크로드가 종료되면 설정 환경을 이미지로 저장해 워크로드를 생성할 때 사용할 수 있습니다. 종료 시 이미지로 저장을 선택한 경우 저장할 이미지 이름과 이미지 태그를 입력합니다.



\- \*\*워크로드 데이터 연결\*\*



&#x20;  !\[3224.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3224.png)



&#x20; - \*\*소스코드\*\*: \*\*+ 소스코드 추가\*\*를 클릭하면 추가창이 나타납니다. 선택한 타입에서 소스코드 목록, Branch, 마운트 경로, 실행 명령어를 입력한 후 \*\*확인\*\*을 클릭해 추가합니다.

&#x20;   - Batch Job에서 Built-in 이미지를 선택한 경우에는 소스코드를 반드시 추가해야 합니다. 소스코드를 추가하면 실행 경로와 명령어가 자동으로 지정됩니다.

&#x20; - \*\*데이터셋 경로\*\*: \*\*+ 데이터셋 경로 추가\*\*를 클릭하면 추가창이 나타납니다. 선택한 타입에서 데이터셋 경로 목록, 마운트 경로를 입력한 후 \*\*확인\*\*을 클릭해 추가합니다.

&#x20; - \*\*모델 경로\*\*: + 모델 경로 추가를 클릭하면 추가창이 나타납니다. 선택한 타입에서 모델 경로 목록, 마운트 경로를 입력한 후 \*\*확인\*\*을 클릭해 추가합니다.

&#x20; - 데이터 추가 시 각 항목의 타입을 선택해 가져올 수 있습니다.

&#x20;   - \*\*워크스페이스\*\*: 공유 리포지토리에 저장된 데이터를 사용합니다.

&#x20;   - \*\*사용자\*\*: 사용자의 개인 리포지토리에 저장된 데이터를 사용합니다.

&#x20;   - \*\*직접 입력\*\*: 소스코드의 경우 신규 생성해 적용합니다. 직접 입력을 선택해 추가한 소스코드는 해당 워크로드에만 적용되는 일회용 자원입니다.

&#x20;   - \*\*데이터 요청\*\*: 사용자가 데이터셋을 요청한 경우 자동차데이터플랫폼(KADaP)의 데이터를 워크로드에 적용할 수 있습니다.



\- \*\*이미지 실행을 위한 실행 명령어 입력\*\*



&#x20;  !\[3236.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3236.png)



&#x20; - \*\*실행 경로\*\*: Batch Job을 선택한 경우 이미지 실행 경로가 자동으로 입력됩니다.

&#x20; - \*\*워크로드 패스워드\*\*: 웹 브라우저로 IDE에 로그인하거나 SSH 명령어로 접속 시 사용해 보안을 강화합니다.

&#x20; - \*\*실행 명령어\*\*: 일반적으로 이미지에는 기본 시작 명령어가 포함되어 있습니다.<br>사용자가 워크로드 생성 시 실행 명령어를 입력하면 이 명령어를 우선 실행합니다.

&#x20;   - 선택한 이미지가 hub 또는 built-in image인 경우에는 사용자가 입력할 수 없습니다.



\- \*\*워크로드의 환경 변수와 포트 입력\*\*



&#x20;  !\[3252.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3252.png)



&#x20; - \*\*환경 변수\*\*: \*\*+ 환경 변수 추가\*\*를 클릭해 워크로드 실행 시 적용할 키와 값을 입력합니다. <br>API 키나 인증 토큰 및 학습 조건을 추가하거나 프레임워크의 동작 방식을 제어할 때 사용할 수 있습니다.

&#x20; - \*\*포트\*\*: \*\*+ 포트 추가\*\*를 클릭해 IDE에 접속 시 필요한 포트 번호와 외부에서 접속할 수 있는 서비스 포트 번호를 입력합니다.

&#x20;   - Built-in Image를 선택한 경우 선택한 항목에 따라 기본 포트는 자동으로 입력됩니다.



\- \*\*워크로드의 시간 예측 파라미터 입력\*\*

&#x20; - Batch Job에서 정확한 학습 시간 예측을 위한 파라미터로 사용자가 학습시키는 모델에 대한 변수입니다. 시간 예측 파라미터 값을 입력하면 예상 학습 수행 시간이 표시됩니다.



&#x20;   !\[3269.png](KADaP\_UserManual\_GPUaaS-web-resources/image/3269.png)



워크로드가 생성되면 워크로드 목록에 표시됩니다. <br>생성이 완료되면 \*\*실행중\*\* 상태로 변경되며 워크로드가 구동됩니다.



\- 구동중인 워크로드는 홈 화면의 GPU 슬롯을 클릭해도 확인할 수 있습니다.



> !\[4513.png](KADaP\_UserManual\_GPUaaS-web-resources/image/4513.png)\*\*참고\*\*

>

> - 관리자가 설정한 GPU 실행 스케줄에 따라 AI 개발 모드와 AI 학습 모드가 실행됩니다. GPU 실행 스케줄은 사용량이나 사용 패턴에 따라 적용 시간 및 적용되는 자원이 달라질 수 있습니다.

> - 워크로드가 종료되면 워크로드의 모든 데이터와 설치 패키지는 자동으로 삭제됩니다. 보관이 필요한 업로드 데이터는 워크로드 종료 전에 마이디스크 폴더에 저장하세요.

>   - 마이디스크 저장 경로: \*\*root\*\* > \*\*자동차데이터플랫폼(KADaP)\*\* > \*\*MyDisk\*\*
