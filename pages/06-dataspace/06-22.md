\## 커넥터 관리하기



데이터 제공자/소비자 역할에 맞는 커넥터를 등록하고 관리할 수 있습니다.



\### 화면 구성



데이터 교환 시스템의 \*\*커넥터 관리\*\* 메뉴에서 커넥터를 등록하고 관리할 수 있습니다.



!\[3646.png](./../../assets/06-dataspace/3646.png)



\### 커넥터 생성하기



커넥터를 등록하려면 다음 순서대로 진행하세요.



1\. 데이터 교환 시스템의 홈 화면에서 \*\*커넥터 관리\*\*를 클릭하세요.

2\. \*\*커넥터 관리\*\* 화면에서 \*\*생성\*\*을 클릭하세요.

3\. 커넥터 생성 화면에서 상세 정보를 입력하고 \*\*생성\*\*을 클릭하세요.

&#x20;  - \*\*커넥터명\*\*: 사용할 커넥터명을 입력합니다.

&#x20;    - 커넥터 유형과 동일하게 제공자일 경우 provider, 소비자인 경우 consumer를 입력하면 쉽게 용도를 확인할 수 있습니다.

&#x20;  - \*\*설명\*\*: 등록할 커넥터 설명을 입력합니다.

&#x20;  - \*\*Dataspace\*\*: 사용할 데이터스페이스를 선택합니다.

&#x20;     - KADaP-X를 선택하면 해당 커넥터가 KADaP-X 데이터스페이스 내의 참여자로 등록되며, 동일한 데이터스페이스 안의 다른 커넥터와 데이터를 교환할 수 있습니다.

&#x20;  - \*\*커넥터 유형\*\*: 등록할 커넥터 유형을 선택합니다.

&#x20;     - 데이터 제공자일 경우 \*\*provider\*\*, 소비자인 경우 \*\*consumer\*\*를 선택합니다.

&#x20;  - \*\*서비스 형식\*\*: 커넥터 서비스 형식을 선택합니다.

&#x20;    - \*\*Self Hosted\*\*: 기업에서 커넥터를 직접 배포하고 운영하는 방식으로 보안이 중요한 경우 선택합니다.

&#x20;    - \*\*CaaS\*\*(Connector as a Service): 자동차데이터플랫폼(KADaP)이 커넥터를 클라우드 환경에서 대신 호스팅하는 방식으로 별도의 서버를 구축하지 않아도 서비스를 이용할 수 있습니다.

&#x20;  - \*\*Base URL\*\*: 커넥터의 컨트롤 플레인 주소로, 다른 참여자가 해당 커넥터에 접속하여 카탈로그 조회, 계약 정의 및 정책 정의 등 데이터 교환에 대한 정보을 확인할 때 사용하는 엔드포인트입니다.

&#x20;  - \*\*데이터플레인 URL\*\*: 커넥터의 데이터 전송 영역 주소로, 계약이 완료된 후 실제 데이터를 주고받는 통로로 사용합니다.

&#x20;  - \*\*API Header\*\*: 커넥터 API를 호출할 때 인증 정보를 전달하는 HTTP 헤더의 이름을 입력합니다.

&#x20;  - \*\*API Value\*\*: 인증에 사용할 비밀번호나 토큰값을 입력합니다.



> !\[4736.png](./../../assets/06-dataspace/4736.png)\*\*참고\*\*

>

> 모든 온보딩 업체는 provider, consumer 커넥터를 등록해야 정상적으로 데이터 공유를 실행할 수 있습니다.



\### 커넥터 정보 확인하기



커넥터 정보 카드의 \*\*커넥터 관리\*\* 메뉴를 클릭해 등록한 커넥터 정보를 확인하고 관리할 수 있습니다.



\- 확인하려는 커넥터 정보 카드의 !\[icon24\_fmt.PNG](./../../assets/06-dataspace/icon24\_fmt.png)를 클릭하면 관리 메뉴가 표시됩니다.

&#x20; - \*\*수정\*\*: 커넥터 수정 화면으로 이동해 커넥터 정보를 수정할 수 있습니다.

&#x20; - \*\*삭제\*\*: 등록한 커넥터를 삭제합니다. 삭제한 커넥터는 복구할 수 없습니다.
