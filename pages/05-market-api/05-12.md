## (개발자) API 배포하기



API의 최초 버전 생성 이후 변경 사항이 발생하면, 신규 버전으로 배포하거나 기존 버전으로 재배포할 수 있습니다.



> ![8991.png](./../../assets/04-market-app/8991.png) **참고**

>

> API의 최초 버전은 생성과 동시에 자동으로 API Gateway에 배포되므로, 별도의 추가 절차를 수행할 필요가 없습니다.



API 상세 페이지 상단의 **API 배포**를 클릭한 후, 버전 선택창에서 **새 버전** 또는 배포할 버전을 선택하세요.



![API 배포 화면](./../../assets/04-market-app/10525.png)



### 신규 버전 배포하기



변경된 내용을 새로운 버전으로 관리할 수 있으며, 이전 버전은 그대로 유지됩니다.<br>여러 버전을 병행 운영해야 하거나 하위 호환성을 유지해야 하는 경우 유용합니다.



### 기존 버전 재배포하기



기존 버전에 변경 사항을 그대로 반영하여 덮어쓰는 방식으로, 이전 버전이 필요 없고 단일 버전만 운영할 때 적합합니다.
