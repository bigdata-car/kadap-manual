# WireGuard VPN 설치 및 활용하기 

## 1. 도입 배경

**자동차 데이터 플랫폼**에서는 기본 `SSL VPN`을 사용하고 있습니다. 하지만 다음과 같은 제약들로 `WireGuard`라는 VPN 방식 도입을 검토 하면서 내용을 정리해 보았습니다. 

- 제약 1 : 일부 네트워크 환경 정책상 SSL VPN방식을 활용 할 수 없는 곳이 있습니다. 
- 제약 2 : (저희가 쓰는 제품의 제약이지만) VPN 접속 프로그램이 윈도우OS는 안정화 되어 있지만, MAC, Linux계열은 불안전 합니다. 
- 제약 3 : (가장 큰 이유)  원격데스크탑 등 GUI 화면 제공을 위해서는 속도가 중요한데, WireGuard대비 속도가 느립니다.

## 2. WireGuard란

다른 VPN 솔루션들과는 달리 두 지점(Peer) 사이의 통신을 암호화하는 프로토콜에 대한 구현체입니다. 따라서, P2P 또는 Mesh형대로도 VPN망을 구성 할 수 있습니다.  

주요 장점은 다음과 같습니다. 
- 빠른 속도 : UDP만을 사용하여 불안정한 연결 환경에서도 속도가 빠름
- 키 방식 인증 : 공개키 암호 방식으로 보안상 안전하고 배포가 간단 합니다. 
- 리눅스 기본 내장 : 커널 5.6부터 기본 재장되어 있어 추가 설치 없이 활용 가능합니다. 
- 오픈소스 : 사용자 추가에 따른 비용 부담이 없습니다. 
- 다양한 클라이언트SW 제공 : 윈도우, 리눅스, 맥, 안드로이드, 아이폰 등 대부분의 OS를 지원 합니다. 

## 2. Peer-to-Peer방식(리눅스 기준)

### 2.1 Node A설정 

#### Node A키 생성 
```
sudo apt install wireguard-tools # 커널 5.6이상은 기본 설치 되어 있으므로 이하 버젼만 진행
wg genkey | tee privatekey | wg pubkey > publickey

# 확인 하기
kadap@tailscale2:~$ cat privatekey
4GsM4YN+E6tHXGbBLrETKfMAwHT7FYNpxUsGVcU5IUE=

kadap@tailscale2:~$ cat publickey
7zLC4IVFnzDh+enLN2iNDd6wXXRJocjdH4GmhReMyQc=

```
### Node A설정 파일 수정
```
sudo vi /etc/wireguard/wg0.conf
  """
  [Interface]
  PrivateKey = 4GsM4YN+E6tHXGbBLrETKfMAwHT7FYNpxUsGVcU5IUE=  #<A의 비밀키>
  Address = 10.0.0.1/32
  ListenPort = 51820
  
  [Peer]
  PublicKey = iNe7YPFkxfrnBR0Jwnp1XXayz0g2ZMB9LUmbZ6wYtG4=  #<B의 공개키>
  Endpoint = 203.0.113.2:51820
  AllowedIPs = 10.0.0.2/32
  PersistentKeepalive = 25
  """
$ sudo systemctl enable wg-quick@wg0.service
$ sudo systemctl start wg-quick@wg0.service
```

## 3. 서버-클라이언트 방식 (윈도우 기준)

> 서버는 네트워크 장비단에서 지원하므로 사용자는 클라이언트SW만 설치 하여 전달 받은 `설정 파일(*.conf)`을 적용하면 됩니다.

설정 파일은 다음과 같은 형식으로 되어 있습니다. 

```
[Interface]
PrivateKey = 2OEhevmSaRGRIgcr/tmvkOqI2cGhH/S  #개인키
Address = 10.131.200.2/24                                          #사용자 IP
DNS = 10.10.10.10, 8.8.8.8                                           #내부 DNS 주소
PostUp = powershell.exe -Command "& { Add-DnsClientNrptRule -Comment 'VPNTunnel' -Namespace '.bigdata-car.kr' -NameServers '10.10.10.10' }"
PostDown = powershell.exe -Command "& { Get-DnsClientNrptRule | where Comment -eq 'VPNTunnel' | foreach { Remove-DnsClientNrptRule -Name $_.Name -Force } }"

[Peer]
PublicKey = 98CVLBlCyUSzUQl6Sj0EnB32pDJIlGmPW   #공개키
PresharedKey = i6KZD/PY84NluIWo89HTXuV71dScpJ   #공유키
AllowedIPs = 10.10.17.0/16                                             #접속 가능 IP 대역, 방화벽단에서 추가 설정되어 있어 변경시 동작 하지 않습니다. 
Endpoint = wireguard.bigdata-car.kr:43146                   #WireGuard 접속 IP:Port

```
