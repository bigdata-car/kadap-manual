



# 2. 데이터 수집 시스템

데이터 수집 시스템(KADaP-X IoT)에서는 자동차 CAN 데이터 및 설비 데이터를 실시간으로 수집하고 전처리를 실행할 수 있습니다. 데이터를 수집하는 장치 상태를 실시간으로 확인하고 제어할 수 있으며, 수집된 로우 데이터에 전처리 모델을 적용해 AAS 표준 기술에 맞는 데이터로 저장합니다.

> ![4736.png](./../../assets/06-dataspace/4736.png) **참고**
>
> 데이터 수집 시스템에는 기관 회원 중 기업 정보 등록이 완료된 사용자만 접속할 수 있습니다.

## 홈 화면 구성

데이터 수집 시스템의 홈 화면은 다음과 같이 구성됩니다. 각 항목별 기능을 설명합니다.

![3326.png](./../../assets/06-dataspace/3326.png)

| 번호 | 항목 | 설명 |
| --- | --- | --- |
| 1 | 데이터 수집 시스템 메뉴 | 데이터 수집 시스템의 메뉴를 선택합니다.<ul><li>**홈**: 홈 화면으로 이동합니다.</li><li>**장치 관리**: 장치를 등록하고 관리할 수 있습니다.</li><li>**프로젝트 관리**: 프로젝트를 등록하고 관리할 수 있습니다.</li><li>**전처리 모델 관리**: 전처리 모델을 등록할 수 있습니다.</li><li>**프로젝트 목록**: 프로젝트 설정 정보와 장치를 확인하고, 프로젝트 데이터를 관리할 수 있습니다.</li>|
| 2 | 내 정보 | 로그인한 사용자 정보를 확인하고 화면 표시 모드를 설정할 수 있습니다.<ul><li>**Dark mode**: 화면 표시 모드를 선택합니다.</li><li>**로그아웃**: 로그인한 계정에서 로그아웃합니다. |
| 3 | 프로젝트 및 장치 상태 | 전체 프로젝트 수와 전체 장치의 상태를 한 눈에 확인할 수 있습니다. |
| 4 | 프로젝트 목록 | 등록한 프로젝트별로 설정 정보와 장치 정보, 프로젝트 데이터를 확인할 수 있습니다.<ul><li>**프로젝트 설정**: 프로젝트 정보와 할당한 장치를 확인할 수 있습니다.</li><li>**프로젝트 데이터 관리**: root 경로에 저장된 데이터를 확인할 수 있습니다.</li><li>장치 정보: 장치 상세 정보와 모니터링 항목을 확인할 수 있으며, 장치별 데이터 및 CAN 뷰어를 설정할 수 있습니다.</li></ul>|
| 5 | 장치 위치 | 등록한 장치의 위치를 지도에서 확인할 수 있습니다. |

## 장치 관리하기

데이터를 수집할 장치를 등록하고 관리할 수 있습니다.

### 화면 구성

데이터 수집 시스템의 **장치 관리** 메뉴에서 장치를 등록하고 관리할 수 있습니다.

![3356.png](./../../assets/06-dataspace/3356.png)

- 장치 관리 항목에서는 장치를 수정하거나 삭제할 수 있습니다.
  - **수정**: 장치 추가 화면으로 이동해 장치 정보를 수정할 수 있습니다.
  - **삭제**: 등록한 장치를 삭제합니다. 삭제한 장치는 복구할 수 없습니다.

### 장치 추가하기

데이터 수집 장치를 등록하려면 다음 순서대로 진행하세요.

1. 데이터 수집 시스템의 홈 화면에서 **장치 관리**를 클릭하세요.
2. 장치 관리 화면에서 **+ 생성**을 클릭하세요.
3. 장치 추가 화면에서 상세 정보를 입력하고 **생성**을 클릭하세요.
- **기본 정보**
  - **장치명**: 추가할 장치 이름을 입력합니다.
  - **설명**: 장치 설명을 입력합니다.
  - **IP**: 장치의 IP 주소를 입력합니다.
- **장치유형 정보**
  - 전처리: 수집된 데이터에 전처리 모델을 적용해 표준 언어로 변환합니다.
  - CAN: 차량 제어 유닛(ECU)에서 발생하는 다양한 정보를 실시간으로 수집합니다.
  - 센서: 제조 현장의 설비 상태나 특정 부품 상태를 측정해 실시간 데이터를 수집합니다.
- **SFTP 접속 정보**
장치의 SFTP(Secure File Transfer Protocol) 접속 정보를 입력합니다. 입력한 SFTP로 수집된 데이터가 암호화되어 전송됩니다.
  - **SFTP PORT**: SFTP 접속 포트를 입력합니다. (기본 포트: 22)
  - **SFTP ID**: SFTP 서버에 접속할 때 필요한 ID 정보를 입력합니다.
  - **SFTP PASSWORD**: SFTP 서버에 접속할 때 필요한 비밀번호를 입력합니다.
- **K8S 접속 정보**
  - **K8S PORT**: 장치의 K8S 접속 포트를 입력합니다. (기본 포트: 6443)
  - 장치의 K8S 클러스터에 접속하기 위한 **kubeconfig** 파일을 업로드합니다. 이 파일에는 API 서버 주소와 인증 정보가 포함되어 있습니다.
- **동기화 정보**
  - **동기화 주기(초)**: 장치와 연결된 클라우드 서버나 저장소와의 동기화 주기를 입력합니다. (기본 동기화 주기: 3600)
- **위치 정보**
    - 검색창에 도로명 주소를 입력하고 **검색**을 클릭하면 지도 위에 위치창이 나타납니다. 위치창의 주소 정보를 확인하고 **선택**을 눌러 장치 위치를 등록합니다.

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> 등록한 장치는 **장치 목록 > 장치 관리** 항목을 클릭해 정보를 수정하거나 삭제할 수 있습니다.

## 프로젝트 관리하기

등록한 장치를 그룹으로 구성하는 프로젝트를 등록하고 관리할 수 있습니다.

### 화면 구성

데이터 수집 시스템의 **프로젝트 관리** 메뉴에서 프로젝트를 등록하고 관리할 수 있습니다.

![3396.png](./../../assets/06-dataspace/3396.png)

- 프로젝트 관리 항목에서는 프로젝트를 수정하거나 삭제할 수 있습니다.
  - **수정**: 프로젝트 편집 화면으로 이동해 프로젝트 정보를 수정할 수 있습니다.
  - **삭제**: 등록한 프로젝트를 삭제합니다. 삭제한 프로젝트는 복구할 수 없습니다.

### 프로젝트 추가하기

프로젝트를 등록하려면 다음 순서대로 진행하세요.

1. 데이터 수집 시스템의 홈 화면에서 **프로젝트 관리**를 클릭하세요.
2. 프로젝트 관리 화면에서 **+ 생성**을 클릭하세요.
3. 프로젝트 추가 화면에서 상세 정보를 입력하고 **생성**을 클릭하세요.
   - **프로젝트명**: 추가할 프로젝트 이름을 입력합니다.
   - **설명**: 프로젝트 설명을 입력합니다.
   - **색상 태그 정보**: 지도에 표시할 프로젝트 색상을 설정합니다.

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> 등록한 프로젝트는**프로젝트 목록 > 프로젝트 관리** 항목을 클릭해 정보를 수정하거나 삭제할 수 있습니다.

## 전처리 모델 관리하기

수집된 데이터 가공 시 적용할 전처리 모델을 등록할 수 있습니다.

### 화면 구성

데이터 수집 시스템의 **전처리 모델 관리** 메뉴에서 전처리 모델을 등록할 수 있습니다.

![3433.png](./../../assets/06-dataspace/3433.png)

### 전처리 모델 추가하기

데이터 전처리 모델을 등록하려면 다음 순서대로 진행하세요.

1. 데이터 수집 시스템의 홈 화면에서 **전처리 모델 관리**를 클릭하세요.
2. 전처리 모델 관리 화면에서 **+ 생성**을 클릭하세요.
3. 전처리 모델 추가 화면에서 모델명을 입력하고 **생성**을 클릭하세요.

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> - 전처리 모델은 추가만 할 수 있습니다. 모델을 삭제하려면 회사의 레포지토리 관리자에게 문의하세요.
> - 데이터 수집 시스템 사용 전 회사 데이터를 관리하는 레포지토리를 설정해야 합니다. 전처리 모델을 추가하면 회사별로 설정된 레포지토리 하위에 경로가 설정됩니다.
>   - 회사 레포지토리와 연동되지 않으며, 전처리 모델 경로 정보만 표시합니다.

## 프로젝트 목록 관리하기

프로젝트별로 장치를 추가해 데이터 수집을 설정하고 관리할 수 있습니다.

### 프로젝트 설정하기

등록한 프로젝트에 장치를 할당해 관리할 수 있습니다.

데이터 수집 프로젝트를 설정하려면 다음 순서대로 진행하세요.

1. 데이터 수집 시스템의 홈 화면에서 **프로젝트 목록**에서 원하는 프로젝트를 클릭하세요.
2. 프로젝트 관리 메뉴에서 **프로젝트 설정**을 클릭하세요.
3. 프로젝트 설정 화면에서 상세 정보를 확인하세요.
   - **프로젝트 정보**: 등록한 프로젝트 정보가 표시됩니다.
   - **위치**: 장치 위치가 지도에 표시됩니다.
   - **프로젝트 장치 목록**: 프로젝트에 할당한 장치 목록을 확인할 수 있습니다.

   ![3466.png](./../../assets/06-dataspace/3466.png)

#### 프로젝트에 장치 추가

프로젝트에 장치를 추가하려면 다음 순서대로 진행하세요.

1. 프로젝트 장치 목록에서 **+ 생성**을 클릭하세요.
2. 프로젝트 장치 추가 화면에서 장치 정보의 **+ 추가**를 클릭하세요.
3. 장치 선택 창에서 등록된 장치 중 원하는 항목을 선택하세요.
   - 데이터 수집 시스템에 등록한 장치만 선택할 수 있으며, 각 장치는 하나의 프로젝트에만 추가할 수 있습니다.
4. 프로젝트 장치 추가 화면에서 **추가**를 클릭하세요.
   - 각 장치 항목에서 상세 정보를 확인하거나 삭제할 수 있습니다.

### 프로젝트 데이터 관리하기 {#프로젝트-데이터-관리하기}

자동차데이터플랫폼(KADaP) 클라우드에 저장된 장치별 데이터를 확인하고 관리할 수 있습니다.

프로젝트 데이터를 관리하려면 다음 순서대로 진행하세요.

1. 데이터 수집 시스템의 홈 화면에서 **프로젝트 목록**에서 원하는 프로젝트를 클릭하세요.
2. 프로젝트 관리 메뉴에서 **프로젝트 데이터 관리**를 클릭하세요.
3. 프로젝트 데이터 설정 화면에서 root 경로에 저장된 데이터를 확인하세요.

   ![3486.png](./../../assets/06-dataspace/3486.png)

  - **파일형식**을 클릭해 필터링 항목을 선택하고 **Apply**를 클릭하면 해당 포맷 파일만 확인할 수 있습니다.
    - 검색창 하단에 선택한 파일형식 항목이 표시됩니다.
    - 파일형식 항목의 **X**를 클릭하거나 **초기화**를 클릭하면 선택한 항목을 해제할 수 있습니다.
  - 폴더 목록 중 **output**폴더 > 장치별 폴더에 전처리 모델로 처리된 데이터가 저장됩니다.
  - **폴더/파일 목록**의![icon29_fmt.PNG](./../../assets/06-dataspace/icon29_fmt.png)를 클릭하면 항목을 추가할 수 있습니다.
    - 폴더 생성 창이 열리면 폴더명을 입력하고 **생성**을 클릭해 추가합니다.
    - 파일 업로드 창이 열리면 업로드할 파일을 선택하고 **업로드**를 클릭해 추가합니다.
  - **폴더/파일 목록**의 ![icon24_fmt.PNG](./../../assets/06-dataspace/icon24_fmt.png)를 클릭하면 선택한 항목을 삭제할 수 있습니다.
    - 파일 항목의 **다운로드**를 클릭하면 해당 파일을 PC로 다운로드할 수 있습니다.

### 장치 상태 정보 확인하기

프로젝트에 할당된 장치를 확인하고 수집된 데이터를 관리할 수 있습니다. 또한 n8n에 로그인해 데이터를 시맨틱 모델에 맞게 가공하도록 설정하며, 수집할 CAN 정보를 설정할 수 있습니다.

프로젝트의 장치 정보를 관리하려면 다음 순서대로 진행하세요.

1. 데이터 수집 시스템의 홈 화면에서 **프로젝트 목록**에서 원하는 프로젝트를 클릭하세요.
2. 프로젝트 관리 메뉴에서 **장치명**을 클릭하세요.
3. 장치 상태 정보 화면에서 탭을 클릭해 상세 정보를 확인하세요.

#### 기본 정보

장치 등록 정보, 모니터링 정보를 확인하고 장치 배포 설정을 할 수 있습니다.

- **장치 정보**: 등록한 장치 정보가 표시됩니다.
- **위치**: 장치 위치가 지도에 표시됩니다.
- **배포 정보**: CAN, 전처리, 센서 유형별로 장치 모델을 추가하고 배포하거나 관리할 수 있습니다.

  ![3529.png](./../../assets/06-dataspace/3529.png)

  - 동작 항목의 **배포**를 클릭하면 적용한 전처리 모델이 실행되며 데이터 수집이 시작됩니다. 배포 상태에서는 **정지**를 클릭해 실행을 중지할 수 있습니다.
  - 배포 모델 관리 항목에서 등록한 정보를 수정하거나 삭제할 수 있습니다.
- **장치 모니터링**: 장치의 하드웨어 및 네트워크 사용량을 모니터링할 수 있습니다.

##### 장치 모듈 추가

장치에 설정할 모듈의 상세 정보를 설정해 추가할 수 있습니다.

장치 모듈 정보를 추가하려면 다음 순서대로 진행하세요.

1. 배포 정보의 **+추가**를 클릭하세요.
2. 장치 모듈 추가 창에서 모듈 선택 항목의 **+선택**을 클릭하세요.
3. 모델 선택 창에서 모듈 유형을 선택하세요. 적용할 모델과 버전을 설정한 후 **+선택**을 클릭하세요.
4. 모듈의 필수 정보와 설정값 정보를 입력하고 **추가**를 클릭하세요.
   - 모듈을 선택한 후에만 필수 정보, 설정값 입력 항목이 표시됩니다.

   ![3545.png](./../../assets/06-dataspace/3545.png)

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> 등록한 장치 모듈은**배포 모델 목록**>**배포 모델 관리** 항목을 클릭해 정보를 수정하거나 삭제할 수 있습니다.

#### 데이터 관리

**데이터 관리** 탭을 클릭하면 장치별로 전처리가 실행되기 전 로우 데이터를 확인할 수 있습니다.
- 데이터 관리 탭에서는 등록한 장치의 HDD에 저장된 정보를 검색할 수 있습니다. 필요 시 파일 업로드/다운로드 및 삭제를 실행할 수 있습니다.
- 데이터 관리 항목에 대한 자세한 정보는 [프로젝트 데이터 관리하기](#프로젝트-데이터-관리하기)를 참고하세요.

#### 노코드

**노코드** 탭을 클릭하면 노코드 워크플로우를 확인할 수 있습니다. 장치에 연결된 노코드 프로그램에 접속해 현재 정보를 가져와 표시합니다.

- n8n 로그인 화면에서 접속 정보를 입력해 상세 정보를 확인할 수 있습니다. n8n 사용 방법은 [https://docs.n8n.io/](https://docs.n8n.io/) 참고하세요.

#### CAN Viewer

**CAN Viewer** 탭을 클릭하면 장치에서 수집할 자동차 관련 CAN 데이터 항목을 설정할 수 있습니다.

![3586.png](./../../assets/06-dataspace/3586.png)

##### CAN 수집 항목 설정

장치에서 수집할 CAN 데이터 매핑 파일을 업로드하고 수집할 항목을 설정할 수 있습니다.

CAN 수집 항목을 설정하려면 다음 순서대로 진행하세요.

1. CAN Viewer 화면에서 **설정**을 클릭하세요.
2. CAN 필터링 창에서 **파일 선택**을 클릭해 CAN 데이터 매핑 파일을 업로드하세요.
   - CAN 데이터 정의 파일은 DCB, CSV 형식 파일만 업로드할 수 있습니다.
3. 업로드가 완료되면 CAN 목록에서 수집할 데이터 항목을 선택하고 **설정**을 클릭하세요.
   - 선택한 장치에 따라 설정할 수 있는 항목이 달라집니다.


# 3. 데이터 교환 시스템

데이터 교환 시스템(KADaP-X Edge)에서는 데이터 자산과 계약 및 정책 정의, 계약 적용 후 데이터 공유를 실행할 수 있습니다. 데이터 공유 시스템의 제공자/소비자 역할에 따라 편리하게 커넥터를 생성하고 관리할 수 있습니다. 또한, 데이터 교환을 위해 필요한 데이터 자산 및 계약, 정책을 정의하고 관리할 수 있습니다.

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> 데이터 교환 시스템에는 기관 회원 중 기업 정보 등록이 완료된 사용자만 접속할 수 있습니다.

## 홈 화면 구성

데이터 교환 시스템의 홈 화면은 다음과 같이 구성됩니다. 각 항목별 기능을 설명합니다.

![3621.png](./../../assets/06-dataspace/3621.png)

| 번호 | 항목 | 설명 |
| --- | --- | --- |
| 1 | 데이터 교환 시스템 메뉴 | 데이터 수집 시스템의 메뉴를 선택합니다.<ul><li>**홈**: 홈 화면으로 이동합니다.</li><li>**커넥터 관리**: 제공자, 소비자 역할의 커넥터를 등록하고 관리할 수 있습니다.</li><li>**공유 자산**: 데이터 교환을 위해 자산을 업로드하고 자산, 정책, 계약을 등록하고 관리할 수 있습니다.</li><li>**디지털 트윈**: 데이터 교환 시 적용되는 AAS 표준을 기반으로 레지스트리 정보를 확인하고 서브 모델을 등록할 수 있습니다.</li><li>**PCF 관리**: 계약 완료 후 PCF(Product Carbon Footprint, 제품 탄소 발자국) 정보를 제공자에게 요청하고 응답 정보를 확인할 수 있습니다.</li><li>**커넥터 목록**: 제공자/소비자가 등록한 커넥터별로 데이터 교환 단계에서 필요한 정보를 등록하고 확인할 수 있습니다.</li></ul> |
| 2 | 조직 정보 | 로그인한 사용자가 속한 회사나 조직 이름이 표시됩니다. |
| 3 | 언어 설정 | 화면에 표시되는 언어를 한국어/영어 중에 선택할 수 있습니다. |
| 4 | 내 정보 | 로그인한 사용자 정보를 확인하고 로그아웃할 수 있습니다. |
| 5 | 데이터 공유 절차 안내 | 데이터 제공자/소비자가 데이터 공유를 위해 실행할 상세 절차를 설명합니다. |

## 커넥터 관리하기

데이터 제공자/소비자 역할에 맞는 커넥터를 등록하고 관리할 수 있습니다.

### 화면 구성

데이터 교환 시스템의 **커넥터 관리** 메뉴에서 커넥터를 등록하고 관리할 수 있습니다.

![3646.png](./../../assets/06-dataspace/3646.png)

### 커넥터 생성하기

커넥터를 등록하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 관리**를 클릭하세요.
2. **커넥터 관리** 화면에서 **생성**을 클릭하세요.
3. 커넥터 생성 화면에서 상세 정보를 입력하고 **생성**을 클릭하세요.
   - **커넥터명**: 사용할 커넥터명을 입력합니다.
     - 커넥터 유형과 동일하게 제공자일 경우 provider, 소비자인 경우 consumer를 입력하면 쉽게 용도를 확인할 수 있습니다.
   - **설명**: 등록할 커넥터 설명을 입력합니다.
   - **Dataspace**: 사용할 데이터스페이스를 선택합니다.
      - KADaP-X를 선택하면 해당 커넥터가 KADaP-X 데이터스페이스 내의 참여자로 등록되며, 동일한 데이터스페이스 안의 다른 커넥터와 데이터를 교환할 수 있습니다.
   - **커넥터 유형**: 등록할 커넥터 유형을 선택합니다.
      - 데이터 제공자일 경우 **provider**, 소비자인 경우 **consumer**를 선택합니다.
   - **서비스 형식**: 커넥터 서비스 형식을 선택합니다.
     - **Self Hosted**: 기업에서 커넥터를 직접 배포하고 운영하는 방식으로 보안이 중요한 경우 선택합니다.
     - **CaaS**(Connector as a Service): 자동차데이터플랫폼(KADaP)이 커넥터를 클라우드 환경에서 대신 호스팅하는 방식으로 별도의 서버를 구축하지 않아도 서비스를 이용할 수 있습니다.
   - **Base URL**: 커넥터의 컨트롤 플레인 주소로, 다른 참여자가 해당 커넥터에 접속하여 카탈로그 조회, 계약 정의 및 정책 정의 등 데이터 교환에 대한 정보을 확인할 때 사용하는 엔드포인트입니다.
   - **데이터플레인 URL**: 커넥터의 데이터 전송 영역 주소로, 계약이 완료된 후 실제 데이터를 주고받는 통로로 사용합니다.
   - **API Header**: 커넥터 API를 호출할 때 인증 정보를 전달하는 HTTP 헤더의 이름을 입력합니다.
   - **API Value**: 인증에 사용할 비밀번호나 토큰값을 입력합니다.

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> 모든 온보딩 업체는 provider, consumer 커넥터를 등록해야 정상적으로 데이터 공유를 실행할 수 있습니다.

### 커넥터 정보 확인하기

커넥터 정보 카드의 **커넥터 관리** 메뉴를 클릭해 등록한 커넥터 정보를 확인하고 관리할 수 있습니다.

- 확인하려는 커넥터 정보 카드의 ![icon24_fmt.PNG](./../../assets/06-dataspace/icon24_fmt.png)를 클릭하면 관리 메뉴가 표시됩니다.
  - **수정**: 커넥터 수정 화면으로 이동해 커넥터 정보를 수정할 수 있습니다.
  - **삭제**: 등록한 커넥터를 삭제합니다. 삭제한 커넥터는 복구할 수 없습니다.

## 공유 자산 관리하기

커넥터를 통해 교환할 데이터를 업로드하고 자산, 정책 및 계약을 정의해 등록할 수 있습니다.

### 자산 업로드하기

AAS 서브모델 템플릿에 맞게 자산을 가공한 후 저장소에 업로드할 수 있습니다.

> ![4736.png](./../../assets/06-dataspace/4736.png)**참고**
>
> 서브모델은 국제 표준인 AAS 포맷에 따라 기술 데이터, 탄소 발자국 등 자산의 특정 정보를 표준화된 구조로 정의한 데이터 단위입니다. 모든 참여자가 동일한 필드 구조, 데이터 타입, 의미 정의에 맞춰 데이터를 작성하므로, 서로 다른 시스템과 조직 사이에서도 데이터 품질과 일관성을 유지하면서 데이터 교환을 할 수 있습니다.

공유할 자산을 업로드하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **공유 자산** > **자산 업로드**를 클릭하세요.
2. 자산 업로드 화면에서 + **생성**을 클릭하세요.
3. 서브모델 선택 창이 나타나면 데이터에 적용할 서브모델을 선택하고 **+ 선택**을 클릭하세요.
   - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)를 클릭하면 서브모델 상세 창에서 버전 및 상세 정보를 확인할 수 있습니다.
4. 선택한 서브모델이 표시되면 자산 정보를 입력하고 **업로드**를 클릭하세요.

   ![3734.png](./../../assets/06-dataspace/3734.png)

   - 서브모델을 선택한 후에만 자산을 입력하거나 업로드할 수 있습니다.
   - 자산 정보를 업로드하기 전에 서브모델의 템플릿이나 작성된 샘플 csv 파일을 다운로드해 상세 항목을 확인합니다. 템플릿에 맞게 자산을 입력해 파일을 업로드하면 자산을 일괄 업로드할 수 있습니다.
   - 자산 정보의 **+ 추가**를 클릭해 직접 세부 항목을 입력하거나 수정할 수 있습니다.

> ![4736.png](./../../assets/06-dataspace/4736.png) **참고**
>
> 자산 업로드가 완료되면 API 정보를 제공합니다. 실제 서비스 배포 시 해당 URL을 적용할 수 있습니다.

### 자산 정의하기

업로드한 자산 데이터를 데이터스페이스 내에서 식별하고 접근할 수 있도록 자산으로 등록합니다. 자산에 고유 ID, 데이터의 위치(엔드포인트), 유형, 설명 등 메타데이터를 정의하면 소비자가 카탈로그에서 해당 자산을 검색하고 확인할 수 있습니다.

#### 화면 구성

데이터 교환 시스템의 **공유 자산**> **자산 정의** 메뉴에서 자산을 정의하고 관리할 수 있습니다.

![3775.png](./../../assets/06-dataspace/3775.png)

- 자산 정의 관리 항목에서는 자산을 수정하거나 삭제할 수 있습니다.
  - **수정**: 자산 편집 화면으로 이동해 자산 정보를 수정할 수 있습니다.
  - **삭제**: 등록한 자산을 삭제합니다. 삭제한 자산은 복구할 수 없습니다.

#### 자산 생성

자산의 메타데이터를 등록하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **공유 자산** > **자산 정의**를 클릭하세요.
2. 자산 정의 화면에서 + **생성**을 클릭하세요.
3. 자산 생성 화면에서 자산의 메타데이터를 입력하고 **생성**을 클릭하세요.
   - **자산**: 등록할 자산 이름과 상세 설명을 입력합니다.
   - **서브모델**: **+ 생성**을 클릭해 자산에 적용한 서브모델을 선택합니다.
     - 자산 업로드 시 선택한 서브모델과 동일한 항목을 선택해야 합니다.
     - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)을 클릭하면 서브모델 창에서 상세 정보를 확인할 수 있으며, ![icon31_fmt.PNG](./../../assets/06-dataspace/icon31_fmt.png)을 클릭하면 서브모델을 다시 선택할 수 있습니다.
   - **소스 유형**: REST, CSV 중 소스 유형을 선택합니다.
     - **REST**: 데이터 교환 시스템이 외부 API에서 직접 데이터를 가져올 수 있도록 연결 정보 및 인증 정보를 설정합니다.
     - **CSV**: csv 파일을 직접 업로드한 경우 해당 파일이 실제 데이터 소스로 등록됩니다.
   - **소스 정보**: 자산 데이터가 업로드된 저장소에 접속할 때 필요한 엔드포인트 정보와 인증 정보를 입력합니다.
     - **엔드포인트**: 외부 API 호출 방식과 데이터를 가져올 외부 API 서버 URL을 입력합니다.
     - **인증**: 외부 API 서버 접속을 위한 인증 유형을 선택하고 Token URL 및 접속 시 필요한 Client ID와 Client Secret을 입력합니다.

   ![3812.png](./../../assets/06-dataspace/3812.png)

### 정책 정의하기

커넥터에 적용할 접근 정책과 사용 정책을 설정합니다. 데이터 제공자는 데이터 사용자 정보와 사용 범위 조건을 정의해 등록합니다. 이 정책을 적용하면 데이터를 공유하면서도 사용자와 사용 범위에 대한 통제권을 유지하는 데이터 주권을 실현할 수 있습니다.

#### 화면 구성

데이터 교환 시스템의 **공유 자산** > **정책 정의** 메뉴에서 데이터 사용 정책을 정의하고 관리할 수 있습니다.

![3824.png](./../../assets/06-dataspace/3824.png)

- 정책 정의 관리 항목에서는 정책을 수정하거나 삭제할 수 있습니다.
  - **수정**: 정책 편집 화면으로 이동해 정책 정보를 수정할 수 있습니다.
  - **삭제**: 등록한 정책을 삭제합니다. 삭제한 정책은 복구할 수 없습니다.

#### 정책 생성

커넥터에 적용할 정책을 등록하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **공유 자산**>**정책 정의**를 클릭하세요.
2. 정책 정의 화면에서 + **생성**을 클릭하세요.
3. 정책 생성 화면에서 정책 상세 정보를 입력하고 **생성**을 클릭하세요.
   - **정책 정보**: 등록할 정책 이름과 상세 설명을 입력합니다.
   - **접근 정책**: 적용할 접근 정책을 추가합니다.
     - **+ 정책 추가**를 클릭해 유형과 값을 추가합니다. **삭제**를 클릭해 추가한 항목을 삭제할 수 있습니다.
   - **사용 정책**: 적용할 사용 정책을 추가합니다.
     - **+ 정책 추가**를 클릭해 유형과 값을 추가합니다. **삭제**를 클릭해 추가한 항목을 삭제할 수 있습니다.

### 계약 정의하기

데이터 제공자는 자산과 정책을 계약으로 묶어 등록합니다. 사전에 등록한 자산과 정책을 설정해 계약을 등록하며, 이 계약 정보가 데이터스페이스 카탈로그에 게시됩니다.

데이터 소비자는 등록된 계약 정보를 확인하고 계약을 요청하게 되며, 계약을 구독한 후에만 실제 데이터에 다운로드할 수 있습니다.

#### 화면 구성

데이터 교환 시스템의 **공유 자산** > **계약 정의** 메뉴에서 계약을 정의하고 관리할 수 있습니다.

![3839.png](./../../assets/06-dataspace/3839.png)

#### 계약 생성

커넥터에 적용할 계약을 등록하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **공유 자산** > **계약 정의**를 클릭하세요.
2. 계약 정의 화면에서 + **생성**을 클릭하세요.
3. 계약 생성 화면에서 계약상세 정보를 입력하고 **생성**을 클릭하세요.
   - **계약 정보**: 등록할 계약 이름과 상세 설명을 입력합니다.
   - **자산 설정**: 계약을 적용할 자산을 추가합니다.
     - **+ 생성**을 클릭해 사전에 등록한 자산 목록 중 선택한 후 **+ 선택**을 클릭합니다.
     - 자산 추가가 완료되면 상세 정보가 표시됩니다. 자산 이름 오른쪽의 ![icon32_fmt.PNG](./../../assets/06-dataspace/icon32_fmt.png)을 클릭하면 추가한 자산을 삭제하고 다시 선택할 수 있습니다.
    - **정책 설정**: 계약을 적용할 정책을 추가합니다.
      - **+ 생성**을 클릭해 사전에 등록한 정책 목록 중 선택한 후 **+ 선택**을 클릭합니다.
      - 정책 추가가 완료되면 상세 정보가 표시됩니다. 정책 이름 오른쪽의 ![icon32_fmt.PNG](./../../assets/06-dataspace/icon32_fmt.png)을 클릭하면 추가한 정책을 삭제하고 다시 선택할 수 있습니다.

## 디지털 트윈 관리하기

데이터스페이스의 레지스트리를 확인하고, 서브모델을 등록하거나 확인할 수 있습니다.

### 레지스트리 확인하기

레지스트리는 데이터스페이스 안에서 특정 자산이 어디에 있는지 알려주는 인덱스 역할을 합니다. 데이터 제공자/소비자는 레지스트리 메뉴에서 다음 항목을 확인할 수 있습니다.

- 데이터 제공자는 자산 업로드 및 자산 정의를 완료한 후 레지스트리에서 해당 디지털 트윈이 정상적으로 등록되었는지 확인할 수 있습니다.
- 데이터 소비자는 데이터스페이스에서 원하는 데이터 자산 정보를 확인할 수 있으며, 해당 자산에 연결된 서브모델과 엔드포인트 정보를 조회해 데이터 교환 여부를 결정합니다.

#### 레지스트리 정보 확인

데이터스페이스에 등록된 레지스트리를 확인하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **디지털 트윈**>**레지스트리**를 클릭하세요.
2. 디지털 트윈 레지스트리 화면에서 상세 정보를 확인할 레지스트리의 ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)을 클릭하세요.

   ![3899.png](./../../assets/06-dataspace/3899.png)

3. 디지털 트윈 쉘 상세 정보 창에서 상세 정보를 확인하세요.

### 서브모델 생성하기

디지털 트윈의 쉘에 연결되는 서브모델을 등록하고 상세 정보를 확인할 수 있습니다.

#### 화면 구성

데이터 교환 시스템의 **디지털 트윈** > **서브모델** 메뉴에서 서브모델을 등록할 수 있습니다.

  ![3915.png](./../../assets/06-dataspace/3915.png)

#### 서브모델 생성

디지털 트윈의 서브모델을 등록하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **디지털 트윈**>**서브모델**을클릭하세요.
2. 서브모델 화면에서 + **생성**을 클릭하세요.
3. 서브모델 생성 화면에서 **Upload file**을 클릭해 파일을 업로드하세요.
   - 서브모델은 json, aasx, xml 형식의 파일만 업로드할 수 있습니다.
4. 업로드가 완료되면 서브모델 정보를 확인하고 **업로드**를 클릭하세요.

#### 서브모델 정보 확인

**디지털 트윈** > **서브모델** 화면에서 상세 정보를 확인할 서브모델의 ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)을 클릭하면 상세 정보 창이 열립니다. 서브모델 상세 정보 창에서 버전 및 시맨틱 ID 등 정보를 확인할 수 있습니다.

## PCF 관리하기

PCF(Product Carbon Footprint)는 제품의 생산 및 유통 과정에서 발생한 탄소 배출량 데이터입니다. 데이터 소비자는 계약이 완료된 후 부품의 PCF 데이터를 제공자에게 요청할 수 있습니다. PCF 데이터는 계약이 완료된 커넥터를 통해 데이터를 요청하고 다운로드할 수 있습니다.

### PCF 요청하기

데이터 교환 계약이 완료되면 PCF 데이터가 필요한 부품의 경우 데이터 제공자에게 해당 데이터 전달을 요청할 수 있습니다.

#### 화면 구성

데이터 교환 시스템의 **PCF 관리** > **PCF 요청** 메뉴에서 PCF 데이터를 요청할 수 있습니다.

![3947.png](./../../assets/06-dataspace/3947.png)

#### PCF 요청 생성

데이터 제공자에게 PCF 데이터를 등록하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **PCF 관리**>**PCF 요청**을클릭하세요.
2. PCF 요청 화면에서 **+PCF 요청**을 클릭하세요.
3. PCF 요청 화면에서 상세 정보와 커넥터 정보를 입력하고 **요청**을 클릭하세요.
   - **PCF 요청 정보**: 계약 정보에 포함되어 있는 BPN, 제품 ID를 확인해 입력하고, 추가로 전달할 메시지를 입력합니다.
   - **커넥터 정보**: PCF 데이터 요청을 전송할 커넥터를 선택합니다.
     - **+ 생성**을 클릭해 사전에 등록한 커넥터 목록 중 선택한 후 **+ 선택**을 클릭합니다. 커넥터 이름 오른쪽의 ![icon32_fmt.PNG](./../../assets/06-dataspace/icon32_fmt.png)을 클릭하면 추가한 커넥터를 삭제하고 다시 선택할 수 있습니다.

### PCF 응답하기

데이터 계약이 완료된 후, 데이터 소비자가 부품의 PCF 데이터를 요청하면 지정된 커넥터로 전달합니다. 데이터 소비자는 제공자가 데이터 전달을 승인하면 커넥터를 통해 다운로드할 수 있습니다.

#### 화면 구성

데이터 교환 시스템의 **PCF 관리** > **PCF 응답** 메뉴에서 전달한 PCF 데이터 정보와 상태를 확인할 수 있습니다.

![3975.png](./../../assets/06-dataspace/3975.png)

#### PCF 응답 확인

데이터 제공자가 전달한 PCF 데이터를 확인하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **PCF 관리**>**PCF 응답**을 클릭하세요.
2. PCF 응답 화면에서 응답 목록의 항목을 확인하세요.
   - 상태 항목에서 **승인 완료**가 표시되면 지정된 커넥터로 PCF 데이터를 다운로드할 수 있습니다.

## 커넥터 목록 관리하기

데이터 교환 시스템에는 사용자는 역할별 커넥터를 생성해 데이터 교환 구독을 실행할 수 있습니다. 데이터 제공자가 커넥터를 생성해 자산을 업로드하고 정책, 계약을 정의해 오퍼링을 등록하면 카탈로그에 게시됩니다. 데이터 소비자는 카탈로그에서 원하는 데이터를 검색해 데이터 교환 계약을 구독합니다. 구독이 승인되면 소비자는 제공자가 업로드한 자산을 다운로드해 이용할 수 있습니다.

커넥터 목록에서는 역할별 커넥터 정보를 확인하고 필요한 업무를 빠르게 실행할 수 있습니다.

### 제공자 (provider)

제공자 커넥터의 주요 업무를 실행하고 진행 상태를 확인할 수 있습니다.

#### 오퍼링 업로드

사전에 등록한 계약을 선택해 커넥터에 업로드할 수 있습니다.

계약을 업로드하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 목록**>**provider [제공자]**>**오퍼링 업로드**를 클릭하세요.
2. 오퍼링 업로드 화면에서 + **생성**을 클릭하세요.
3. 계약 선택 창이 나타나면 적용할 계약을 선택하고 **+ 선택**을 클릭하세요.
   - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)를 클릭하면 서브모델 상세 창에서 버전 및 상세 정보를 확인할 수 있습니다.
4. 선택한 계약의 자산 및 정책 정보가 표시되면 상세 내용을 확인하고 **업로드**를 클릭하세요.
   - 계약 이름 오른쪽의 ![icon32_fmt.PNG](./../../assets/06-dataspace/icon32_fmt.png)을 클릭하면 추가한 계약을 삭제하고 다시 선택할 수 있습니다.

#### 계약 관리

소비자와의 계약 정보 및 상태를 확인하고 구독을 관리할 수 있습니다.

계약을 확인하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 목록**>**provider [제공자]**>**계약 관리**를 클릭하세요.
2. 계약 관리 화면에서 계약 상세 정보를 확인하세요.

   ![4010.png](./../../assets/06-dataspace/4010.png)

   - **Negotiation ID**: 계약을 식별하는 고유 번호로 시스템이 자동으로 부여합니다.
   - **Counter Party Address**: 계약을 요청한 데이터 소비자의 커넥터 주소 정보입니다. 이 커넥터 정보로 데이터가 실제로 전달됩니다.
   - **Asset ID**: 계약에 포함된 자산ID로 제공자가 자산을 정의할 때 등록한 이름을 표시합니다.
   - **Status**: 해당 계약의 상태를 표시합니다.
     - FINALIZED 상태인 경우에만 계약이 최종 체결되어 데이터 교환이 진행됩니다.
   - **Signing Date**: 계약이 최종 확정된 날짜를 표시합니다. 날짜 정보는 체결된 계약에만 표시됩니다.
   - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)를 클릭하면 계약 상세 창에서 자산 및 정책 정보를 확인할 수 있습니다.
   - ![icon24_fmt.PNG](./../../assets/06-dataspace/icon24_fmt.png)를 클릭하면 계약 관리 메뉴에서 구독을 취소하거나 거절할 수 있습니다.
      - **구독 취소**: 계약이 체결된 후 제공자가 구독을 취소할 수 있습니다. 계약 완료 후 소비자가 사용 정책을 위반했거나, 더 이상 데이터 제공이 어려운 경우 사용합니다.
     - **구독 거절**: 소비자가 계약을 요청했을 때 구독을 거절할 수 있습니다. 정책 조건에 맞지 않거나 신뢰할 수 없는 소비자인 경우 계약이 체결되기 전에 사용할 수 있습니다.

#### 전송 이력

계약이 완료된 후 데이터 소비자의 데이터 요청 이력 및 전송 결과를 확인할 수 있습니다.

전송 이력을 확인하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 목록**>**provider [제공자]**>**전송 이력**을 클릭하세요.
2. 전송 이력 화면에서 전송 이력 및 결과를 확인하세요.

   ![4051.png](./../../assets/06-dataspace/4051.png)

    - **ID**: 데이터 전송을 식별하는 ID로 전송 상세 내역을 조회할 때 사용됩니다.
    - **상태**: 데이터 전송의 처리 결과를 보여줍니다.
      - **성공/실패** 항목의 숫자로 실제 데이터의 정상 전송 여부를 확인할 수 있습니다.
      - **갱신/삭제** 항목의 숫자로 데이터의 변경 및 삭제 여부를 확인할 수 있습니다.
    - **시작일**: 소비자가 다운로드를 요청한 시점으로 데이터 전송이 시작된 일시입니다.
    - **종료일**: 데이터 전송이 완료된 일시입니다. 시작일과 종료일을 비교해 전송에 소요된 시간을 확인할 수 있습니다.
    - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)를 클릭하면 전송된 데이터의 상세 정보를 확인할 수 있습니다.

### 소비자 (consumer)

소비자 커넥터의 주요 업무를 실행하고 처리 상태를 확인할 수 있습니다.

#### 카탈로그 조회

카탈로그에서 데이터 제공자가 등록한 계약 정보를 확인할 수 있습니다.

카탈로그에서 제공자 정보를 조회하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 목록**>**consumer [소비자]**>**카탈로그 조회**를 클릭하세요.
2. 카탈로그 조회 화면에서 데이터 제공자의 BPN 정보를 입력하고 **검색**을 클릭하세요.
   - 제공자의 BPN 정보는 **데이터 교환 시스템 포탈** > **파트너 네트워크** 메뉴에서 확인할 수 있습니다.
3. 입력한 데이터 제공자의 계약 정보가 표시되면 원하는 계약을 선택해 구독을 신청하세요.

소비자가 구독을 신청하면 제공자에게 내역이 전송됩니다. 제공자가 구독을 승인하면 계약이 체결되어 계약 정보를 확인하고 실제 데이터를 다운로드할 수 있습니다.

#### 계약 관리

제공자와의 계약 정보 및 상태를 확인하고 구독을 관리할 수 있습니다.

계약을 확인하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 목록**>**consumer [소비자]**>**계약 관리**를 클릭하세요.
2. 계약 관리 화면에서 계약 상세 정보를 확인하세요.

   ![4081.png](./../../assets/06-dataspace/4081.png)

   - **Negotiation ID**: 계약을 식별하는 고유 번호로 시스템이 자동으로 부여합니다.
   - **Counter Party Address**: 계약을 요청한 데이터 소비자의 커넥터 주소 정보입니다. 이 커넥터 정보로 데이터가 실제로 전달됩니다.
   - **Asset ID**: 계약에 포함된 자산ID로 제공자가 자산을 정의할 때 등록한 이름을 표시합니다.
   - **Status**: 해당 계약의 상태를 표시합니다.
     - FINALIZED 상태인 경우에만 계약이 최종 체결되어 데이터 교환이 진행됩니다.
   - **Signing Date**: 계약이 최종 확정된 날짜를 표시합니다. 날짜 정보는 체결된 계약에만 표시됩니다.
    - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)를 클릭하면 계약 상세 창에서 자산 및 정책 정보를 확인할 수 있습니다.
   - ![icon24_fmt.PNG](./../../assets/06-dataspace/icon24_fmt.png)를 클릭하면 계약 관리 메뉴에서 데이터를 다운로드하고 구독을 취소할 수 있습니다.
     - **다운로드**: 계약이 정상 체결되면 제공자에게 데이터 다운로드를 요청할 수 있습니다.
     - **구독 취소**: 계약이 체결된 후 소비자가 구독을 취소할 수 있습니다.

#### 다운로드 이력

계약이 완료된 후 데이터 다운로드 이력 및 전송 결과를 확인할 수 있습니다.

다운로드 이력을 확인하려면 다음 순서대로 진행하세요.

1. 데이터 교환 시스템의 홈 화면에서 **커넥터 목록**>**consumer [소비자]**>**다운로드 이력**을 클릭하세요.
2. 다운로드 이력 화면에서 다운로드 이력 및 결과를 확인하세요.

   ![4115.png](./../../assets/06-dataspace/4115.png)

   - **ID**: 데이터 전송을 식별하는 ID로 다운로드 상세 내역을 조회할 때 사용됩니다.
   - **제공자 URL**: 데이터 제공자의 저장소 URL 정보를 표시합니다.
   - **상태**: 데이터 전송의 처리 결과를 보여줍니다.
     - **성공/실패** 항목에서 실제 데이터의 정상 전송 여부를 확인할 수 있습니다. 
     - **갱신/삭제** 항목에서 데이터의 변경 및 삭제 여부를 확인할 수 있습니다.
   - **시작일**: 데이터 다운로드가 시작된 일시입니다.
   - **종료일**: 데이터 다운로드가 완료된 일시입니다. 시작일과 종료일을 비교해 다운로드에 소요된 시간을 확인할 수 있습니다.
   - ![icon30_fmt.PNG](./../../assets/06-dataspace/icon30_fmt.png)를 클릭하면 다운로드한 데이터의 상세 정보를 확인할 수 있습니다.
