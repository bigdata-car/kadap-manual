
# 마켓 플레이스 KADaP MARKET

자동차 데이터 기반의 분석 및 개발에 필요한 다양한 데이터뿐만 아니라, APP과 API 형태의 상품이나 서비스를 간편하게 구매하고 활용할 수 있습니다.

# 마켓 플레이스 소개

마켓 플레이스는 KADaP 내에서 운영되는 데이터 기반 거래 공간으로, 기업과 기관이 데이터를 공유하거나 판매하고, 필요한 데이터를 구매하는 등 데이터 상품의 거래와 서비스 환경을 제공합니다.

마켓 플레이스는 SaaS 마켓 플레이스와 API 마켓 플레이스로 구성되어 있고, 각 마켓 플레이스의 서비스 구성과 주요 특징은 다음과 같습니다.

## 주요 기능과 특징

### SaaS 마켓 플레이스

- **APP 검색 및 탐색 기능 제공**

  다양한 자동차 기술 분야의 APP을 카테고리별로 분류하여 쉽게 찾아볼 수 있습니다.

- **APP 이용 신청**

  원하는 APP을 선택하여 이용 신청하고, 승인 후 바로 사용할 수 있습니다. 무료 APP은 이용 신청을 하면 바로 사용 가능하고, 유료 APP은 개발자 승인 후 사용할 수 있습니다.

- **포인트 기반 과금**

  포인트(무료 APP)·포인트 플러스(유료 APP)를 통해 사용 시간에 따라 차감됩니다.

- **APP 등록 및 게시 기능 제공**

  개발자는 자체 개발한 APP을 등록하고, 관리자 승인 후 마켓에 공개할 수 있습니다.

- **승인 기반 운영 체계**

  관리자의 검토 및 승인 절차를 통해 신뢰할 수 있는 APP만 마켓에 공개됩니다.

- **대시보드**

  사용 중인 APP 목록과 이용 상태를 한눈에 확인할 수 있습니다.

### API 마켓 플레이스

- **API 신청 · 구독 · 호출 기능 제공**

  사용자가 원하는 API를 선택해 신청하고, 구독 후 바로 호출할 수 있습니다.

- **표준화된 서비스 구조**

  API 문서와 요청/응답 규격, 인증 체계를 통일된 방식으로 구성해 사용자 편의성을 높였습니다.

- **승인 기반 보안 체계**

  개발자 및 관리자의 승인 절차를 거쳐 안전하게 API를 이용할 수 있는 환경을 제공합니다.

- **사용 통계 및 구독 관리 대시보드**

  API 호출 현황, 응답 상태, 에러 비율, 사용 신청 내역 등 API 사용 현황을 직관적으로 파악할 수 있습니다.

# SaaS 마켓 플레이스

SaaS 마켓 플레이스는 자동차데이터플랫폼(KADaP) 내에서 운영되는 APP 거래 플랫폼으로, 자동차 기술 분야의 다양한 APP을 카테고리별로 검색하고 간편하게 이용 신청하여 사용할 수 있습니다. 또한 개발자는 자체 개발한 APP을 등록하고 관리자 승인 후 마켓에 공개할 수 있습니다.

### APP

마켓에 등록된 APP은 카테고리별로 분류되어 있어 쉽게 찾아볼 수 있습니다. 또한 검색창에 키워드를 입력하여 원하는 APP을 바로 검색할 수도 있습니다. 검색 결과는 리스트 또는 카드 형식으로 표시할 수 있으며, 최신순 또는 인기순으로 정렬할 수 있습니다.

- **전체 APP**: 마켓에 등록된 전체 APP 목록입니다.
- **운영체제(OS)**: 운영체제 관련 소프트웨어 APP 목록입니다.
- **데이터 처리**: 데이터 수집·가공·분석 관련 APP 목록입니다.
- **소프트웨어 개발**: 개발 도구, 코딩 환경, 버전 관리 등의 APP 목록입니다.
- **자동차 산업 특화**: 자동차 제조·부품·자율주행 등 산업 특화 APP 목록입니다.
- **사무환경DX화**: 업무 환경의 디지털 전환을 지원하는 사무용 APP 목록입니다.

- **승인 관리**: 사용자가 신청한 APP 이용 요청을 승인하거나 반려합니다.

## APP 사용하기

사용자는 SaaS 마켓 플레이스에 등록된 APP을 검색하고, 사용 신청할 수 있습니다.

### APP 사용 신청하기

APP 사용하려면 다음 순서대로 진행하세요.

#### APP 검색

1. APP 마켓 플레이스 검색창 또는 **APP** 메뉴에서 원하는 APP을 검색하세요.
   - 키워드 검색, 카테고리 선택, 정렬 옵션(인기순, 최신순)을 활용해 원하는 APP을 빠르게 찾을 수 있습니다.

   ![APP 검색 화면](KADaP_UserManual_Market-web-resources/image/12464.png)

2. 검색 결과에서 원하는 APP을 클릭하세요.
   - APP 상세 페이지로 이동하며, 해당 APP의 상세 정보, 도커 정보, 요금제 정보를 확인할 수 있습니다.

#### APP 사용 신청

1. APP 상세 페이지에서 **이용 신청**을 클릭하세요.

   ![이용 신청 화면](KADaP_UserManual_Market-web-resources/image/12459.png)

2. **구매 신청** 화면에서 다음 항목을 설정하고 **신청**을 클릭하세요.

   ![구매 신청 화면](KADaP_UserManual_Market-web-resources/image/12775.png)

   - **보유 포인트**: 현재 보유한 포인트와 포인트 플러스를 확인합니다.
   - **시간당 예상 차감**: 선택한 APP의 시간당 예상 차감 포인트를 확인합니다.
   - **예상 사용 시간**: 사용 예정 시간을 입력하여 총 차감 포인트를 예상할 수 있습니다.
   - **권장 사양**: CPU, Memory, DISK 권장 사양을 확인합니다.
   - **보유 자원**: 현재 할당된 CPU, Memory, DISK 자원을 확인합니다.
   - **mydisk 연동**: 포털의 마이디스크와 연동할지 선택합니다.
   - **Scale to Zero**: 사용자가 없을 때 절전 모드로 전환합니다. 포인트 차감이 50% 감소합니다.
   - **포인트 플러스**: 보유 자원 제약 없이 사용합니다. 관리자가 지정한 포인트 플러스 단가로 차감됩니다.

   APP 이용 신청이 완료되었고, 해당 APP 개발자의 승인 절차가 진행됩니다.

> ![8991.png](KADaP_UserManual_Market-web-resources/image/8991.png) **참고**
>
> 무료 APP은 이용 신청 즉시 사용할 수 있습니다. 유료 APP은 개발자의 승인 이후에 이용할 수 있습니다.

#### APP 이용 신청 상태 확인

**마이페이지** > **APP 신청 현황**에서 신청한 APP의 목록과 상태를 확인할 수 있습니다.

![APP 신청 현황 화면](KADaP_UserManual_Market-web-resources/image/12440.png)

- **사용/실행 대기**: APP 승인은 완료되었으나, 사용자가 아직 실행하지 않은 건수를 표시합니다.
- **사용/실행**: APP을 정상적으로 사용할 수 있는 건수를 표시합니다.
- **정지**: 포인트 부족 또는 사용자 요청에 의해 APP이 정지된 건수를 표시합니다.
- **서비스 목록**
  - **실행**: APP을 사용할 수 있습니다.
  - **실행 대기**: APP 승인은 완료되었고, 사용자 실행 대기 상태입니다.
  - **승인 대기**: 개발자의 이용 승인 대기 상태입니다.

> ![8991.png](KADaP_UserManual_Market-web-resources/image/8991.png) **참고**
>
> 사용 신청한 APP의 승인 여부는 이메일을 통해 안내됩니다.

### APP 실행하기

무료 APP은 이용 신청 즉시 사용할 수 있고, 유료 APP은 개발자의 승인 이후에 이용할 수 있습니다.

APP을 실행하려면 다음 순서대로 진행하세요.

1. **마이페이지** > **APP 신청 현황**에서 사용할 APP을 클릭하세요.

   ![APP 신청 현황에서 APP 선택](KADaP_UserManual_Market-web-resources/image/12842.png)

2. 상세 화면에서 실행정보를 확인하세요.
   - **실행정보** 탭에서 APP의 도커 이미지 정보, 접속 URL, 실행 포트 구성(SSH·HTTP·HTTPS), CPU 및 메모리 할당 현황 등 APP 구동에 필요한 상세 정보를 확인할 수 있습니다.

   ![실행정보 탭 화면](KADaP_UserManual_Market-web-resources/image/12955.png)

3. **실행정보** 탭의 Knative URL을 클릭하여 해당 APP에 접속하세요.

   ![Knative URL 접속 화면](KADaP_UserManual_Market-web-resources/image/12837.png)

> ![8991.png](KADaP_UserManual_Market-web-resources/image/8991.png) **참고**
>
> 유료 APP은 사용 승인이 완료된 후 Knative URL이 활성화됩니다.

### APP 실행 모니터링 및 제어하기

APP을 SaaS 마켓 플레이스에 등록하여 사용자에게 공개할 수 있습니다. 등록된 APP은 승인 과정을 거쳐 공개되며, 사용자는 해당 APP에 대해 이용 신청을 진행할 수 있습니다.

APP을 SaaS 마켓 플레이스에 등록하려면 다음 순서대로 진행하세요.

1. **APP 등록관리** > **APP 등록**을 클릭하세요.

   ![APP 등록 메뉴 화면](KADaP_UserManual_Market-web-resources/image/13447.png)

2. APP 기본 정보를 입력하세요.

   ![APP 기본 정보 입력 화면](KADaP_UserManual_Market-web-resources/image/13468.png)

3. APP 기술 정보를 입력하세요.

   ![APP 기술 정보 입력 화면](KADaP_UserManual_Market-web-resources/image/13486.png)

   - **입력 방식**
   
      APP 설정 정보를 입력하는 방식을 선택합니다. **직접입력** 선택 시 각 항목을 개별 필드에 입력하고, **YAML 입력** 선택 시 Knative 기반의 YAML 코드 입력창이 표시되어 설정 정보를 한 번에 입력할 수 있습니다.

   - **프록시 여부**
   
      APP에 프록시 서버 사용 여부를 설정합니다.
  
   - **도커 이미지 이름**
   
      배포할 APP의 도커 이미지 이름을 입력합니다.
   
   - **이미지 태그**
   
      배포할 도커 이미지의 버전 태그를 입력합니다.
   
   - **도커 이미지 URL**
   
      Docker Hub 등 도커 이미지의 상세 정보를 확인할 수 있는 링크를 입력합니다.
  
   - **Command**
   
      컨테이너 실행 시 수행할 명령어를 입력하며, 여러 개의 명령어는 세미콜론(;)으로 구분합니다.
  
   - **실행포트**
   
      APP이 사용할 포트명, 포트 번호, 프로토콜을 설정하며, 포트는 최대 15자로 입력합니다.
  
   - **환경 변수**
   
      컨테이너 실행 시 필요한 환경 변수의 변수명과 변수값을 설정합니다.
  
   - **권장 사양**
   
      APP 실행에 권장되는 CPU(0.5~2 Core) 및 메모리(1~8192 Mi) 사양을 설정합니다.

4. APP 이용 요금 및 컴퓨팅 자원 요금을 설정하세요.

   시간당 차감 포인트는 (APP 요금 + 컴퓨팅 자원 요금 + GPU 자원 요금) × 사용 시간으로 산정됩니다.

   ![APP 요금 설정 화면](KADaP_UserManual_Market-web-resources/image/13514.png)

   - **APP 요금 (개발자 보상)**
   
     무료 또는 유료로 설정할 수 있습니다. 유료 선택 시 시간당 포인트를 입력하고, 포인트 유형을 Point 또는 Point+ 중에서 선택할 수 있습니다.

   - **컴퓨팅 자원 요금 (운영자 보상)**
   
      APP 실행에 사용되는 CPU·MEM·DISK 자원에 대한 시간당 요금을 입력합니다.
   
   - **GPU 지원 (GPU 제공자 보상)** 
   
     GPU 자원 사용 여부를 설정하며, Yes 선택 시 GPU 자원 요금이 포인트 산정에 포함됩니다.

   ![요금제 내용 입력 화면](KADaP_UserManual_Market-web-resources/image/13568.png)

6. 기술지원 정보를 입력하세요.
   - APP 이용 중 발생할 수 있는 문의나 장애에 대한 기술지원 방법 등을 입력합니다. 작성한 내용은 사용자의 APP 상세 화면 내 기술지원 정보 항목에 표시됩니다.

### API

마켓에 등록된 API는 카테고리별로 분류되어 있어 쉽게 찾아볼 수 있습니다. 또한 검색창에 키워드를 입력하여 원하는 API를 바로 검색할 수도 있습니다. 검색 결과는 리스트 또는 카드 형식으로 표시할 수 있으며, 최신순 또는 인기순으로 정렬할 수 있습니다.

- **전체 API**: 마켓에 등록된 전체 API 목록입니다.
- **서비스 API**: 입력된 데이터를 처리하고, AI 기반의 분석 및 예측 기능을 수행하는 API 목록입니다.
- **자동차산업특화**: 차량 번호 기반 이력 조회, 운전면허증 확인, 차량 이미지 인식 등 자동차 서비스 현장에서 활용 가능한 실무 중심의 API 목록입니다.
- **AI/알고리즘**: 다양한 대형 언어 모델(LLM)과 이미지 분석, 텍스트 음성 변환(TTS) 등 AI 기반 기능을 손쉽게 연동할 수 있는 API 목록입니다.

### API Key 관리하기

API 마켓 플레이스에서는 승인된 모든 API 테스트에 사용할 수 있는 공용 API Key를 제공합니다. 공용 API Key는 여러 API에서 공통으로 사용되기 때문에 노출이나 오용되지 않도록 주의하시기 바랍니다.

#### 재발급

API Key가 노출되었거나 보안 상의 이유로 새 Key가 필요한 경우, **재발급** 기능을 이용해 새로운 API Key를 생성할 수 있습니다. 재발급 후 기존 Key는 즉시 사용 중단되며, 새로 발급된 Key로만 API 호출이 가능합니다.

![재발급 화면](KADaP_UserManual_Market-web-resources/image/9353.png)

#### 구독취소

API Key는 사용자가 구독취소 전까지 지속적으로 사용할 수 있습니다. API가 불필요한 경우 **구독취소**를 통해 해당 API 사용을 해지할 수 있습니다. 구독취소 후에는 API Key가 비활성화되며, API 호출이 불가능합니다.

![구독취소 화면](KADaP_UserManual_Market-web-resources/image/9385.png)

#### 보안 주의사항

API Key는 외부에 노출될 경우 부정 사용 또는 오용 위험이 있으므로 다음 보안 사항을 준수해야 합니다.

- API Key는 외부에 공유하거나 저장소(깃허브 등)에 공개하지 않습니다.
- 노출이 의심되는 경우 즉시 재발급하여 기존 Key를 폐기합니다.
- 클라이언트 애플리케이션 등에서 직접 노출되지 않도록 서버 환경에서 안전하게 관리합니다.
- 불필요한 Key는 구독 취소하여 보안 위험을 최소화합니다.

> ![8991.png](KADaP_UserManual_Market-web-resources/image/8991.png) **참고**
>
> API 등록과 관리는 **개발자 센터**에서 수행하며, **개발자 센터**를 이용하려면 개발자 권한이 필요합니다.<br>개발자 권한 신청은 [개발자 등록](#api-개발자-등록)를 참고하세요.

API를 등록하려면 다음 순서대로 진행하세요.

  설정이 완료된 API를 선택한 환경에 반영하여 실제 서비스에 배포합니다.

  마켓 플레이스에 API를 노출하기 위한 게시를 요청합니다. 게시된 API는 관리자 승인 후 마켓 플레이스에 공개됩니다.

### 메서드 등록하기

메서드(Method)는 API 요청 시 서버가 수행해야 할 동작을 지정하는 규칙입니다.<br>데이터 조회(GET), 생성(POST), 수정(PUT/PATCH), 삭제(DELETE) 등의 메서드를 사용하며, 각 메서드는 요청 목적을 명확히 구분해 서버가 올바르게 동작하도록 합니다.

메서드 등록은 API의 엔드포인트별 동작 방식을 정의하는 단계입니다. 각 메서드에서 경로, 타겟 경로, 타겟 서버 그룹 등을 설정할 수 있으며, 이를 통해 API 호출 시 어떤 요청이 수행될지 결정됩니다.

   - **경로**: 엔드포인트의 경로를 입력합니다. 사용자에게 노출되는 경로로 실제 API의 경로와 일치하지 않아도 무방합니다.
   - **메서드**: GET, POST, PUT, DELETE 등의 요청 방식을 선택합니다.
   - **API KEY 필요**: API 호출 시 API KEY를 헤더 또는 파라미터로 전달하여 권한을 검증합니다.
   - **타겟 경로**: 타겟 경로에 요청을 전달할 실제 서버의 경로를 입력합니다.
   - **타겟 서버 그룹**: API에 설정된 그룹을 그대로 사용할 수도 있고, 필요시 개별적으로 설정할 수 있습니다.

#### 파라미터 설정

요청 파라미터는 API 호출 시 클라이언트가 전달하는 입력값을 정의하는 단계입니다. 필수 여부, 데이터 타입, 배열 여부 등을 설정할 수 있으며, 이는 API 요청을 검증하고 일관성있는 동작을 보장하기 위해 필요합니다.

1. 요청 파라미터 정보를 입력하세요.

   - **이름/설명**: 실제 요청 시 사용될 파라미터 Key 값과 설명을 입력합니다.
   - **타입**: 데이터 타입(String, Number)을 선택합니다.

   - 요청 바디는 필수 입력 항목이 아닙니다. 필요한 경우에만 컨텐츠 타입, JSON 스키마 등을 입력하여 요청 구조와 사용 방법을 명확히 정의할 수 있습니다.
